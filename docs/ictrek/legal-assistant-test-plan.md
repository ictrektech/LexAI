# 法律助手测试计划

本文定义法律助手第一版核心闭环测试计划，用于验证法律知识库问答、检索引用、合同审查、知识图谱基础能力和无依据拒答等场景是否达到初步可用标准。

本计划不是正式法律意见质量背书，也不是完整上线验收清单。它的目标是建立一套可执行、可复现、可汇报的测试框架，并支持后续补充初步完成度测试结果。

参考资料：

- [法律小助手用户使用指南](USERGUIDE.md)
- [法律知识图谱配置指引](legal-knowledge-graph-config.md)
- [QA 数据集工具](../../dataset/qa_dataset.py)

## 1. 测试目标

验证法律助手在核心法律场景下是否能够完成从文档入库到问答输出的闭环，并能给出可追溯、可复核的回答。

本测试计划用于回答：

- 法律助手是否能完成从文档入库到问答的核心闭环；
- 回答是否基于知识库内容，而不是模型自由发挥；
- 引用来源是否准确、可追溯；
- 合同审查场景是否能输出风险点和修改建议；
- 无依据问题是否能正确拒答；
- 知识图谱是否能按法律模板完成基础抽取。

## 2. 测试范围

### 范围内

- 新建法律知识库；
- 上传法律法规、案例、合同类文档；
- 文档解析、分块、向量化状态检查；
- 快速问答 RAG 检索；
- 法律法规问答；
- 案例事实问答；
- 合同条款审查；
- 引用来源准确性检查；
- 无依据问题拒答；
- 多轮追问；
- 法律知识图谱冒烟测试；

### 范围外

- 大规模压力测试；
- 全浏览器兼容性测试；
- 正式法律意见准确性背书；
- 全量法规覆盖；
- IM 集成完整验收；
- 生产级安全渗透测试。

## 7. 初步完成度统计方法

初测建议按模块加权计算：

| 模块 | 权重 |
| --- | ---: |
| 知识库与文档入库 | 15% |
| 法律问答准确性 | 30% |
| 检索与引用 | 25% |
| 合同审查 | 15% |
| 无依据拒答 | 10% |
| 知识图谱冒烟 | 5% |

完成度计算：

```text
模块得分 = 模块通过用例数 / 模块总用例数 * 模块权重
总完成度 = 所有模块得分之和
```

## 10. 风险与限制

- 法律回答需要人工复核，测试通过不代表可直接作为正式法律意见；
- 脚本自动判分和命中统计不能完全衡量法律准确性；
- 测试结果依赖知识库文档质量、分块质量和模型配置；
- 模型输出存在随机性，关键用例建议重复测试；
- 知识图谱是增强能力，第一版只做冒烟验证，不作为核心问答通过的硬性前提；
- 合同审查结果应重点检查是否能指出风险、引用依据、给出可操作建议，而不是追求唯一标准答案。

## 11. 文档检查清单

- Test plan 覆盖目标、范围、数据、指标、用例和完成度算法；
- 每个 P0 用例都有明确输入、前置条件和预期结果；
- 用例与现有知识库、快速问答、合同审查和知识图谱流程一致；
- 指标以人工评估为主，脚本自动判分和命中统计为辅；
- 初测可按 30 条样本执行，并能填入通过情况、完成度和问题清单。
