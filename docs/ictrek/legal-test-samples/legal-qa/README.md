# 法律法规问答测试样例

本目录提供法律助手法律法规问答专项测试材料，用于覆盖法条直接问答、跨条款归纳、引用准确性和回答结构完整性。

本测试只验证知识库问答是否能基于已入库法规作答，不构成正式法律意见。

## 文件结构

| 文件 | 用途 |
| --- | --- |
| [test-cards.md](test-cards.md) | 法规问答测试卡、问题、期望要点和通过标准 |
| [test-cases.json](test-cases.json) | 自动测试脚本使用的结构化用例配置 |
| [run_legal_qa_tests.py](run_legal_qa_tests.py) | 自动执行法规问答用例并生成结果报告 |

## 前置条件

- 已有「法律条文」知识库，且法规文档已完成解析、分块和向量化。
- 第一轮样例默认使用当前法律条文库中的法规文档，不新建知识库。
- 建议法律条文库至少包含《中华人民共和国民法典》《中华人民共和国劳动合同法》《中华人民共和国公司法》《中华人民共和国个人信息保护法》《中华人民共和国消费者权益保护法》等文档。

当前样例按以下已验证文档设计：

- `中华人民共和国民法典_20200528.docx`
- `中华人民共和国劳动合同法_20121228.docx`
- `中华人民共和国劳动合同法实施条例_20080918.docx`
- `中华人民共和国公司法_20231229.docx`
- `中华人民共和国个人信息保护法_20210820.docx`
- `中华人民共和国电子商务法_20180831.docx`
- `中华人民共和国消费者权益保护法_20131025.docx`
- `最高人民法院关于审理买卖合同纠纷案件适用法律问题的解释_20201229.doc`

## 自动执行

使用 API Key：

```bash
python3 docs/ictrek/legal-test-samples/legal-qa/run_legal_qa_tests.py \
  --host "http://localhost:8080" \
  --api-key "$WEKNORA_API_KEY" \
  --law-kb-id "<法律条文知识库ID>"
```

本地单用户模式也可以使用自动登录：

```bash
python3 docs/ictrek/legal-test-samples/legal-qa/run_legal_qa_tests.py \
  --host "http://localhost:8080" \
  --auto-setup \
  --law-kb-id "<法律条文知识库ID>"
```

如需指定快速问答智能体：

```bash
python3 docs/ictrek/legal-test-samples/legal-qa/run_legal_qa_tests.py \
  --host "http://localhost:8080" \
  --auto-setup \
  --law-kb-id "<法律条文知识库ID>" \
  --endpoint agent \
  --agent-id "<法规问答智能体ID>"
```

输出目录默认为 `results/<时间戳>/`，包含：

- `summary.md`：通过数、法规问答可用率和每条用例的机器判断；
- `results.json`：每条用例的完整回答、引用、事件和命中明细。

脚本退出码说明：

- `0`：所有用例机器检查通过；
- `1`：至少 1 条用例需要人工复核或执行失败。

## 判定口径

单条法规问答用例满足以下条件时，记为通过：

- 命中测试卡中的核心答案要点，允许同义表达；
- 回答中能出现目标法律、条文编号、制度名称或关键法条内容；
- 返回标准引用 `knowledge_references`，或答案中明确引用相关法律条文；
- 回答没有明显偏离问题或编造不存在的法规名称。

## 结果统计

本样例默认执行 10 条法规问答用例。

```text
法律法规问答可用率 = 通过用例数 / 10
```

自动判定只作为初筛；最终质量仍需人工复核引用位置和法条准确性。
