# 法律知识图谱测试卡

本文件固定法律知识图谱样例的输入问题、预期节点、预期关系、引用检查点和通过标准。测试时不要把“预期节点”和“预期关系”提供给模型。

## 通用输出要求

每条问题都要求模型：

- 使用图谱化结构输出，允许 Markdown 表格、缩进列表、Mermaid graph、JSON-like 节点/边列表或显式“节点/关系”清单；
- 至少包含“节点”“关系”“依据/来源”三类信息；
- 节点应覆盖法律概念、主体、行为、条件、责任、救济或案例要素；
- 关系应体现“适用、构成、导致、承担、引用、支持、救济”等连接；
- 对知识库未覆盖的内容明确说明依据不足，不编造法条、案号或裁判观点。

## LKG-001 劳动合同解除概念关系图

| 字段 | 内容 |
| --- | --- |
| 用例编号 | LKG-001 |
| 图谱类型 | 法律概念关系图 |
| 用户问题 | 请围绕“劳动合同解除”建立法律概念关系图，区分协商解除、劳动者单方解除、用人单位单方解除、经济补偿和违法解除责任，并列出依据。 |
| 应选择知识库 | 法律条文库 |
| 预期节点 | 劳动合同解除、协商解除、劳动者单方解除、用人单位单方解除、经济补偿、违法解除、继续履行、赔偿金 |
| 预期关系 | 劳动合同解除-包括-协商解除；劳动合同解除-包括-劳动者单方解除；劳动合同解除-包括-用人单位单方解除；解除情形-可能产生-经济补偿；违法解除-导致-继续履行或赔偿金 |
| 应引用依据 | 《劳动合同法》解除、经济补偿、违法解除责任相关条款 |
| 输出格式要求 | 节点表 + 关系列表，或 Mermaid graph，并附依据说明 |
| 通过标准 | 命中劳动合同解除核心分类、经济补偿和违法解除责任节点；至少列出 3 条有效关系；引用劳动合同法或相关条文来源 |
