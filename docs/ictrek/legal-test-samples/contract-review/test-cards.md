# 合同审查测试卡

本文件固定每份合同的测试输入、期望风险点、引用检查点和通过标准。测试时不要把“期望风险点”直接提供给模型，只上传合同并发送测试问题。

## 通用输出要求

P0 问题用于专项风险识别：

```text
请站在甲方立场审查这份合同的违约责任条款，输出风险点、依据和修改建议。
```

P1 问题用于完整报告和长答案检查：

```text
请按高/中/低风险输出完整合同审查报告，并给出修改建议。报告至少包含审查摘要、总体风险等级、风险清单、重点条款修改、缺失条款建议、待确认事项和免责声明。
```

## CONTRACT-001 设备采购合同

| 项目 | 内容 |
| --- | --- |
| 合同文件 | `contracts/contract-001-equipment-purchase.md` |
| 合同类型 | 设备采购合同 |
| 审查立场 | 甲方 / 买方 |
| P0 重点 | 违约责任、交付、验收、付款 |
| P1 重点 | 完整风险报告 |

期望风险点：

- 付款节点过早，80% 预付款发生在发货前，甲方资金安全弱。
- 验收标准不清，仅写“基本符合使用要求”，缺少测试指标、整改期限和复验机制。
- 逾期交付违约金过低，且以 5% 封顶，不足以覆盖停产或替代采购损失。
- 质量责任期限只有 30 日，且甲方逾期未提异议视为全部合格。
- 争议解决地为空白，后续管辖不确定。

引用检查点：

- 第 3 条付款方式；
- 第 4 条交付；
- 第 5 条验收；
- 第 7 条质量保证；
- 第 9 条违约责任；
- 第 12 条争议解决。

通过标准：

- P0 至少命中“违约金过低/封顶过低”和“验收标准不清”两个核心风险；
- P1 至少输出 4 项风险，并给出可直接落地的条款修改方向；
- 引用能定位到合同条款，并能结合现有法律条文或案例依据说明理由。

## CONTRACT-002 技术服务合同

| 项目 | 内容 |
| --- | --- |
| 合同文件 | `contracts/contract-002-technical-service.md` |
| 合同类型 | 技术服务合同 |
| 审查立场 | 甲方 / 委托方 |
| P0 重点 | 违约责任、成果交付、知识产权、验收 |
| P1 重点 | 完整风险报告 |

期望风险点：

- 服务范围写得宽泛，缺少里程碑、人员投入、交付清单和验收标准。
- 甲方需在合同签署后 3 日内支付 90% 服务费，付款早于主要成果交付。
- 成果知识产权归乙方所有，甲方仅有“内部使用权”，不符合委托开发常见需求。
- 逾期交付仅退还未完成部分费用，没有违约金和损失赔偿机制。
- 保密期限只有 1 年，且未覆盖源代码、技术文档、经营数据等资料。

引用检查点：

- 第 2 条服务内容；
- 第 3 条项目周期；
- 第 4 条费用与付款；
- 第 5 条成果交付与验收；
- 第 6 条知识产权；
- 第 8 条保密；
- 第 9 条违约责任。

通过标准：

- P0 至少命中“逾期责任不足”和“付款早于成果交付”；
- P1 应覆盖知识产权归属、验收标准、保密和责任限制；
- 修改建议应包括里程碑验收、付款比例调整、成果权属或授权范围重写。

## CONTRACT-003 SaaS 订阅服务合同

| 项目 | 内容 |
| --- | --- |
| 合同文件 | `contracts/contract-003-saas-subscription.md` |
| 合同类型 | SaaS 订阅服务合同 |
| 审查立场 | 甲方 / 订阅方 |
| P0 重点 | 服务中断、数据安全、责任限制、单方变更 |
| P1 重点 | 完整风险报告 |

期望风险点：

- 服务可用性只承诺“尽商业合理努力”，没有 SLA、故障响应时间和补偿机制。
- 乙方可单方调整功能和价格，甲方只有继续使用视为同意的被动接受机制。
- 数据备份周期不明确，合同终止后 7 日删除数据，缺少导出和迁移安排。
- 乙方责任总额封顶为最近 1 个月服务费，且排除间接损失，风险过度转嫁给甲方。
- 数据安全条款缺少泄露通知时限、审计配合、分包商管理和合规要求。

引用检查点：

- 第 2 条服务内容；
- 第 4 条服务可用性；
- 第 5 条数据处理；
- 第 6 条费用与变更；
- 第 8 条责任限制；
- 第 9 条合同终止。

通过标准：

- P0 至少命中“责任上限过低”和“服务中断补偿不足”；
- P1 应覆盖数据安全、单方变更、数据返还/删除和 SLA；
- 修改建议应给出 SLA 数值、泄露通知时限、数据导出期限或责任上限调整方向。

## CONTRACT-004 场地租赁及合作协议

| 项目 | 内容 |
| --- | --- |
| 合同文件 | `contracts/contract-004-lease-cooperation.md` |
| 合同类型 | 场地租赁及合作协议 |
| 审查立场 | 甲方 / 承租及合作方 |
| P0 重点 | 解除权、押金、交付状态、违约责任 |
| P1 重点 | 完整风险报告 |

期望风险点：

- 出租方可因“经营需要”提前 15 日单方解除，未约定补偿。
- 场地交付状态和消防、产权、授权文件未列明，甲方经营合规风险高。
- 押金退还条件不清，乙方可单方抵扣“其他费用”。
- 甲方逾期付款违约金较高，乙方逾期交付或无法经营的责任较轻。
- 收益分成口径不清，未说明税费、退款、平台费用是否扣除。

引用检查点：

- 第 2 条场地交付；
- 第 3 条租金及分成；
- 第 4 条押金；
- 第 6 条经营资质；
- 第 8 条提前解除；
- 第 9 条违约责任。

通过标准：

- P0 至少命中“乙方单方解除权失衡”和“违约责任不对等”；
- P1 应覆盖押金、交付状态、经营资质和收益分成；
- 修改建议应包括解除补偿、交付附件、押金返还期限和违约责任对等化。

## CONTRACT-005 长篇框架采购合同

| 项目 | 内容 |
| --- | --- |
| 合同文件 | `contracts/contract-005-long-framework.md` |
| 合同类型 | 长篇框架采购合同 |
| 审查立场 | 甲方 / 采购方 |
| P0 重点 | 违约责任、订单履行、验收、责任上限 |
| P1 重点 | 长答案完整性和完整审查报告 |

期望风险点：

- 框架合同与订单优先级不清，后续订单冲突时解释风险高。
- 乙方可在原材料上涨超过 5% 时单方调价，缺少甲方确认和退出权。
- 分批交付延误只按单批货款 0.02%/日计违约金，并以单批货款 3% 封顶。
- 甲方未在 5 日内验收即视为合格，复杂设备验收期过短。
- 乙方责任总额以最近 3 个月订单金额为限，且不赔停产、替代采购、数据恢复等损失。
- 合同缺少供应链中断、关键部件替代、召回、备件保障和业务连续性安排。

引用检查点：

- 第 2 条文件优先级；
- 第 4 条订单与价格；
- 第 5 条交付；
- 第 6 条验收；
- 第 9 条质量与售后；
- 第 11 条违约责任；
- 第 12 条责任限制；
- 第 15 条不可抗力与供应链中断。

通过标准：

- P0 至少命中“逾期责任过低”和“责任上限过低/排除关键损失”；
- P1 报告必须包含审查摘要、总体风险等级、风险清单、重点条款修改、缺失条款建议、待确认事项；
- P1 不应在中途截断，并至少输出 6 项风险。
