# 多轮追问测试卡

## MTF-001 劳动合同解除

- 用例编号：MTF-001
- 场景：用人单位解除劳动合同，逐轮补充试用期、严重违纪、未签书面劳动合同等事实。
- 轮次问题：
  - 第 1 轮：公司想解除一名入职 5 个月员工的劳动合同，目前只知道绩效不达标，没有书面考核记录。违法解除风险大吗？
  - 第 2 轮：补充一下，这名员工还在试用期内，公司能否因为“不符合录用条件”直接解除？
  - 第 3 轮：如果后来发现他多次伪造报销单，公司规章制度中明确列为严重违纪，结论会变化吗？
  - 第 4 轮：再补充一个事实：公司一直没有和他签书面劳动合同，这会影响解除风险和赔偿责任吗？
- 应选择知识库：法律条文库。
- 每轮预期承接点：
  - 第 1 轮：入职 5 个月、绩效不达标、无书面考核记录。
  - 第 2 轮：承接前轮“解除劳动合同”和“绩效不达标”，识别新增“试用期”“录用条件”。
  - 第 3 轮：承接试用期解除背景，识别新增“伪造报销单”“严重违纪”“规章制度”。
  - 第 4 轮：承接前述解除理由，识别新增“未签书面劳动合同”及双倍工资等独立风险。
- 每轮预期新增判断：
  - 第 1 轮：仅凭绩效不达标且缺少书面考核依据，违法解除风险较高，应补充制度、考核和沟通证据。
  - 第 2 轮：试用期解除仍需证明不符合录用条件，不能仅以试用期为由随意解除。
  - 第 3 轮：若严重违纪事实、规章制度和送达程序成立，解除依据可能转向严重违纪，风险相对下降。
  - 第 4 轮：未签书面合同不当然阻止解除，但会构成独立违法风险，并产生双倍工资、责任叠加和证据风险。
- 应引用依据：《劳动合同法》关于试用期解除、严重违纪解除、违法解除赔偿、未签书面合同责任的条款。
- 禁止行为：把试用期当作任意解除；忽略无考核记录；忽略严重违纪对结论的影响；把未签合同错误理解为劳动关系不存在；给出“必胜”“一定合法”等确定性结论。
- 通过标准：4 轮中多数轮次能承接前文事实，识别新增事实并调整风险判断，至少 3 轮给出劳动合同法依据或引用。

## MTF-002 消费者权益保护

- 用例编号：MTF-002
- 场景：消费者退货退款，逐轮补充网购、线下购买、定制商品、超过七日等事实。
- 轮次问题：
  - 第 1 轮：消费者买了一件商品后不满意，想要求商家退货退款，法律上一般怎么判断？
  - 第 2 轮：如果这是电商平台网购，消费者收到货第 3 天提出退货，是否适用七日无理由退货？
  - 第 3 轮：如果同样是第 3 天，但商品是按消费者尺寸定制的西装，还能无理由退货吗？
  - 第 4 轮：如果不是网购，而是线下门店购买，并且已经超过 7 天，还能要求退货退款吗？
- 应选择知识库：法律条文库。
- 每轮预期承接点：
  - 第 1 轮：商品不满意、退货退款的一般判断。
  - 第 2 轮：承接退货退款，识别新增网购、收到货第 3 天。
  - 第 3 轮：承接网购和第 3 天，识别新增定制商品。
  - 第 4 轮：承接退货退款问题，识别事实变化为线下购买且超过 7 天。
- 每轮预期新增判断：
  - 第 1 轮：需区分质量问题、合同约定、平台规则和无理由退货适用条件。
  - 第 2 轮：网购通常可在收到商品 7 日内无理由退货，但需考虑例外。
  - 第 3 轮：定制商品通常属于七日无理由退货例外，除非有质量问题或另有承诺。
  - 第 4 轮：线下购买不当然适用七日无理由退货；超过 7 天后通常需看质量问题、约定或经营者承诺。
- 应引用依据：《消费者权益保护法》七日无理由退货及例外，《电子商务法》平台责任可作为辅助。
- 禁止行为：把所有商品都认定可七日无理由退货；忽略定制商品例外；把线下购买等同网购；超过七日仍给无来源确定性结论。
- 通过标准：4 轮中多数轮次能承接“退货退款”主线并按网购、定制、线下、超过七日调整结论，至少 3 轮给出消费者权益保护法依据或引用。

## MTF-003 买卖合同违约责任

- 用例编号：MTF-003
- 场景：买卖合同延期交付责任，逐轮补充违约金过低、未约定验收标准、已有催告记录等事实。
- 轮次问题：
  - 第 1 轮：供应商迟延交付设备 20 天，买方想追究违约责任，通常可以主张哪些责任？
  - 第 2 轮：合同约定逾期违约金每天 100 元，但买方停产损失明显更高，能否要求提高或另行赔偿？
  - 第 3 轮：合同没有写清验收标准，供应商说已经交货，买方说不能验收，这会影响迟延交付责任吗？
  - 第 4 轮：买方已经发过两次书面催告并保留邮件记录，这对解除合同或主张损失有什么帮助？
- 应选择知识库：法律条文库、法律案例库。
- 每轮预期承接点：
  - 第 1 轮：供应商迟延交付设备 20 天、买方追究违约责任。
  - 第 2 轮：承接迟延交付，识别新增每天 100 元违约金和停产损失更高。
  - 第 3 轮：承接迟延交付与违约责任，识别新增验收标准不清、是否已交货争议。
  - 第 4 轮：承接迟延交付、验收争议，识别新增两次书面催告和邮件证据。
- 每轮预期新增判断：
  - 第 1 轮：可主张继续履行、违约金、赔偿损失、解除条件下的解除等。
  - 第 2 轮：约定违约金过低时可请求适当增加，损失赔偿仍受实际损失和可预见规则约束。
  - 第 3 轮：验收标准不清会影响是否构成合格交付、违约起算和举证责任，应补充验收依据。
  - 第 4 轮：书面催告有助于证明履行迟延、给付宽限期、解除条件和损失扩大防止。
- 应引用依据：《民法典》违约责任、损失赔偿、违约金调整、合同解除，以及买卖合同司法解释中检验和验收相关规则。
- 禁止行为：忽略违约金过低调整规则；把未约定验收标准直接等于买方无权拒收；忽略催告证据；无依据认定一定解除成功。
- 通过标准：4 轮中多数轮次能承接迟延交付主线，并随违约金、验收、催告新增事实调整责任分析，至少 3 轮给出民法典或买卖合同依据。

## MTF-004 个人信息保护

- 用例编号：MTF-004
- 场景：收集手机号和身份证号是否合法，逐轮补充同意、必要范围、敏感个人信息等事实。
- 轮次问题：
  - 第 1 轮：一个 App 注册时要求用户填写手机号和身份证号，这样收集个人信息是否合法？
  - 第 2 轮：如果 App 已经弹窗取得用户同意，是否就一定合法？
  - 第 3 轮：如果 App 只是普通资讯阅读应用，却强制收集身份证号才能浏览文章，结论会变化吗？
  - 第 4 轮：身份证号是否属于敏感个人信息？如果涉及敏感个人信息，处理规则有什么更高要求？
- 应选择知识库：法律条文库。
- 每轮预期承接点：
  - 第 1 轮：App 注册、手机号、身份证号、收集个人信息。
  - 第 2 轮：承接手机号和身份证号收集，识别新增取得同意。
  - 第 3 轮：承接已同意的讨论，识别新增普通资讯阅读、强制收集身份证号、超出必要范围。
  - 第 4 轮：承接身份证号收集，识别新增敏感个人信息和更高处理要求。
- 每轮预期新增判断：
  - 第 1 轮：合法性需看目的、必要性、告知同意、公开处理规则等，不能抽象认定合法。
  - 第 2 轮：同意不是唯一条件，仍需满足合法、正当、必要和最小必要原则。
  - 第 3 轮：普通资讯阅读强制收集身份证号可能超出必要范围，不应因拒绝非必要信息而拒绝基本服务。
  - 第 4 轮：身份证号通常可涉及敏感个人信息，应有特定目的、充分必要性、单独同意和影响告知等要求。
- 应引用依据：《个人信息保护法》个人信息处理原则、告知同意、最小必要、敏感个人信息处理规则。
- 禁止行为：把取得同意等同于绝对合法；忽略必要性和最小范围；否认身份证号风险；无依据给出合规背书。
- 通过标准：4 轮中多数轮次能承接个人信息收集主线，并按同意、必要性、敏感信息新增事实调整合规结论，至少 3 轮给出个人信息保护法依据。

## MTF-005 案例追问

- 用例编号：MTF-005
- 场景：先要求提炼案例争点，再追问相似案例和裁判理由变化。
- 轮次问题：
  - 第 1 轮：请从法律案例库中选择一个消费者权益保护相关案例，提炼案件事实、争议焦点和裁判结果。
  - 第 2 轮：承接刚才这个案例，请说明法院支持或不支持消费者主张的核心裁判理由。
  - 第 3 轮：如果有相似案例，请比较它们在事实要素或裁判理由上的差异；如果知识库没有相似案例，也请明确说明。
- 应选择知识库：法律案例库，必要时辅助法律条文库。
- 每轮预期承接点：
  - 第 1 轮：消费者权益保护案例、事实、争议焦点、裁判结果。
  - 第 2 轮：承接刚才选定的案例，不应重新随机换案，继续说明法院裁判理由。
  - 第 3 轮：承接前案的事实和争点，比较相似案例或说明知识库没有检索到相似案例。
- 每轮预期新增判断：
  - 第 1 轮：能从案例库中提取案情、争点和结果，避免空泛普法。
  - 第 2 轮：能围绕同一案例解释法院支持或不支持的理由，并区分事实认定和法律适用。
  - 第 3 轮：能比较相似案例的关键事实、争点或裁判理由差异；无相似案例时不编造。
- 应引用依据：案例名称、案号、法院观点或裁判理由；可辅助引用消费者权益保护法相关条款。
- 禁止行为：第 2 轮更换案例却不说明；编造案号或相似案例；把案例事实当作普遍规则；无依据总结裁判趋势。
- 通过标准：3 轮中至少 2 轮能围绕同一案例承接追问，继续给出案例依据；第 3 轮不编造相似案例。
